# Configuration

Customize every aspect of the VIP System to match your server's needs.

<Info>
  All configuration is in `shared/config.lua`. Restart the resource after making changes.
</Info>

---

## Discord Role Configuration

Configure which Discord roles grant VIP access.

<Steps>
  <Step title="Get Role IDs">
    Enable Developer Mode in Discord → Right-click roles → Copy ID
  </Step>
  <Step title="Add to Config">
    Paste role IDs into the config file
  </Step>
  <Step title="Assign Roles">
    First role = VIP, Second role = VIP+
  </Step>
</Steps>

```lua shared/config.lua
Config.Discord = {
    vipRoles = {
        '123456789012345678', -- VIP role ID
        '987654321098765432'  -- VIP+ role ID
    }
}
```

<Warning>
  If you only have one VIP tier, use the same role ID twice.
</Warning>

## Framework Settings

Choose your framework or let the system auto-detect it.

```lua shared/config.lua
Config.Framework = 'auto' -- 'auto', 'esx', 'qbcore', 'standalone'
```

<Tabs>
  <Tab title="Auto-Detect" icon="wand-magic-sparkles">
    **Recommended** - Automatically detects your framework

    ```lua
    Config.Framework = 'auto'
    ```

    Detection order: ESX → QBCore → QBX → Standalone
  </Tab>

  <Tab title="Manual" icon="hand">
    Manually specify your framework:

    ```lua
    -- For ESX
    Config.Framework = 'esx'

    -- For QBCore/QBX
    Config.Framework = 'qbcore'

    -- For no framework
    Config.Framework = 'standalone'
    ```
  </Tab>
</Tabs>

## Skin Menu Settings

Configure which skin/clothing menu to use.

```lua shared/config.lua
Config.SkinMenu = 'auto' -- Auto-detect or specify manually
```

**Supported Menus**:
- illenium-appearance
- qb-clothing
- esx_skin
- skinchanger
- fivem-appearance
- codem-appearance
- ag_appearance

<Tip>
  Leave as `'auto'` to automatically detect available skin menus.
</Tip>

## VIP Features Configuration

Customize each VIP feature's price, cooldown, and required tier.

### Feature Structure

```lua
Config.Features = {
    feature_name = {
        enabled = true,           -- Enable/disable feature
        price = 500,             -- Cost in money
        cooldown = 300,          -- Cooldown in seconds
        vipLevel = 'vip',        -- 'vip' or 'vip_plus'
        title = 'Feature Name',  -- Display name
        description = 'Text',    -- Description shown in menu
        icon = 'heart'           -- Font Awesome icon
    }
}
```

## VIP Shop Configuration

Configure the exclusive VIP shop with custom items and pricing.

```lua shared/config.lua
Config.Shop = {
    enabled = true,
    title = 'VIP Shop',
    subtitle = 'Exclusive items for VIP members',
    currency = 'money',  -- 'money', 'bank', 'black_money'
    items = {
        vip = {
            -- VIP tier items
        },
        vip_plus = {
            -- VIP+ tier items
        }
    }
}
```

### Adding Shop Items

<Tabs>
  <Tab title="VIP Items" icon="star">
    Items available to VIP members:

    ```lua
    vip = {
        {name = 'bread', label = 'Bread', price = 25, count = 5, icon = 'bread-slice'},
        {name = 'water', label = 'Water Bottle', price = 15, count = 3, icon = 'tint'},
        {name = 'bandage', label = 'Bandage', price = 150, count = 2, icon = 'band-aid'}
    }
    ```
  </Tab>

  <Tab title="VIP+ Items" icon="crown">
    Items for VIP+ members (better prices + exclusive):

    ```lua
    vip_plus = {
        -- Discounted versions
        {name = 'bread', label = 'Bread (VIP+ Discount)', price = 12, count = 10, icon = 'bread-slice'},
        {name = 'water', label = 'Water (VIP+ Discount)', price = 8, count = 5, icon = 'tint'},
        
        -- Exclusive items
        {name = 'lockpick', label = 'Lockpick (VIP+ Exclusive)', price = 500, count = 3, vipLevel = 'vip_plus', icon = 'key'},
        {name = 'repairkit', label = 'Repair Kit (VIP+ Exclusive)', price = 750, count = 1, vipLevel = 'vip_plus', icon = 'tools'}
    }
    ```
  </Tab>
</Tabs>

**Item Structure**:
- `name` - Item spawn name (must exist in your inventory)
- `label` - Display name in shop
- `price` - Cost to purchase
- `count` - How many you get
- `icon` - Font Awesome icon name
- `vipLevel` - (Optional) Required VIP tier

<Warning>
  Make sure all items exist in your framework's inventory system!
</Warning>

## Quick Travel Configuration

Set up teleport locations for VIP+ members.

```lua shared/config.lua
Config.QuickTravel = {
    enabled = true,
    vipLevel = 'vip_plus',  -- Required tier
    locations = {
        {name = 'Hospital', coords = vector3(295.87, -1446.94, 29.97), price = 200},
        {name = 'City Hall', coords = vector3(-544.85, -204.13, 38.22), price = 200},
        {name = 'Airport', coords = vector3(-1037.17, -2737.73, 20.17), price = 300},
        {name = 'Beach', coords = vector3(-1393.84, -1020.32, 7.68), price = 250}
    }
}
```

### Adding Locations

<Steps>
  <Step title="Get Coordinates">
    Stand at the location in-game and run `/getcoords` (or similar)
  </Step>
  <Step title="Add to Config">
    ```lua
    {name = 'Location Name', coords = vector3(x, y, z), price = 200}
    ```
  </Step>
  <Step title="Set Price">
    Adjust price based on location desirability
  </Step>
</Steps>

<Tip>
  Popular locations can have higher prices. Consider making spawn points cheaper.
</Tip>

## Menu Configuration

Customize the VIP menu appearance and keybind.

```lua shared/config.lua
Config.Menu = {
    title = 'VIP Menu',
    subtitle = 'Exclusive features for VIP members',
    position = 'top-right',  -- 'top-right', 'top-left', etc.
    keybind = {
        name = 'vip_menu',
        key = 'F5',          -- Keybind to open menu
        description = 'Open VIP Menu'
    }
}
```

**Position Options**:
- `top-right`
- `top-left`
- `center`

**Common Keybinds**:
- `F5`, `F6`, `F7` (function keys)
- `HOME`, `INSERT` (special keys)

## Testing Your Configuration

After configuring:

<Steps>
  <Step title="Save Config">
    Save `shared/config.lua`
  </Step>
  <Step title="Restart Resource">
    Run `restart jx_vip` in console
  </Step>
  <Step title="Test Features">
    Join server, press F5, test each feature
  </Step>
  <Step title="Adjust as Needed">
    Fine-tune prices and cooldowns based on feedback
  </Step>
</Steps>

<Tip>
  Start with higher prices and lower them based on player feedback. It's easier to reduce prices than increase them!
</Tip>
